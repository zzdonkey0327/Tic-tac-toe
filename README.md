# 卡牌井字對戰

以 Windows Forms 製作的棋牌類遊戲，結合井字棋、卡牌手牌管理、共用牌庫、鬼牌機制與簡易 AI 對戰。專案符合課程 HW2「棋牌類遊戲」作業方向，使用圖片牌面顯示，並加入開始、勝利、失敗等音效回饋。

## 專案簡介

本遊戲將傳統井字棋改造成卡牌對戰形式。玩家與 AI 共用同一個牌庫，輪流抽牌或出牌到 3×3 棋盤上；當三格皆由同一方佔領時，即形成有效連線，並以該線上三張牌的點數總和計分，最後比較總分決定勝負。

遊戲牌組不是標準 52 張撲克牌，而是由數值牌 1–5 各兩張，再加上一張鬼牌所組成，共 11 張。鬼牌不能直接放到空格，而是用來移除棋盤上既有的牌，讓對局除了搶位置之外，還多了反制與節奏控制的策略性。

AI 不是隨機亂下，而是會依序考慮完成連線、使用鬼牌拆除玩家威脅、阻擋玩家成線、依位置優先出牌，位置優先順序為中間格、四個角、最後才是邊格，因此對戰過程有基本的攻防判斷。

## 遊戲特色

- **卡牌化井字棋玩法**：將棋盤佔位與手牌管理結合，讓每一步不只考慮位置，也要考慮手上的牌值與出牌時機。
- **共用牌庫設計**：雙方共用 11 張牌，抽牌會改變後續選項，使每一局都有隨機性。
- **鬼牌特殊效果**：抽到鬼牌後可移除場上既有牌，增加逆轉與干擾效果。
- **成線計分機制**：不是只比單一三連線，而是檢查全部 8 條線，所有有效線分數都會累加。
- **語音與音效回饋**：可搭配 `startplay`、`winplay`、`loseplay` 等語音檔，在開局與結算時提供更完整的遊戲氛圍。
- **自訂主題 UI**：介面以玩家區、AI 區、牌庫區、操作區與對戰棋盤分區顯示，並以不同色系區分雙方陣營。
## 遊戲規則

1. 遊戲開始後，玩家先手，雙方輪流進行回合。[2]
2. 每回合只能執行一種主要行動：抽一張牌、放一張數值牌，或使用一張鬼牌。
3. 玩家手牌上限為 3 張；牌庫尚有剩餘且手牌未滿時可以抽牌。
4. 數值牌只能放到棋盤上的空格，且放置後該格會標示陣營與點數。
5. 鬼牌不能當作一般數值牌使用；玩家使用鬼牌時，可移除場上任一已放置的牌，AI 則優先用鬼牌拆除玩家的重要威脅。
6. 棋盤填滿，或雙方皆無法再抽牌與出牌時，遊戲進入結算。
7. 結算時檢查 8 條線（3 橫、3 直、2 斜），若同一條線的三格都屬於同一方，則該線成立，分數為三張牌點數總和；多條有效線可累加，總分高者獲勝。

## 操作方式

- 點擊 **抽一張牌**：從共用牌庫抽牌到自己的手牌區。
- 點擊 **玩家手牌**：選擇要出的數值牌或鬼牌。
- 再點擊 **棋盤格子**：若選的是數值牌，需點空格放置；若選的是鬼牌，需點已有牌的格子進行移除。
- 點擊 **重新開始**：立即重設牌局並重新洗牌。

## 介面說明

程式主畫面由多個區塊組成，方便玩家快速理解目前牌局狀態。

- **遊戲資訊區**：顯示目前回合、牌庫剩餘張數與狀態訊息。
- **對戰棋盤**：3×3 的主要對戰區，會顯示牌面圖片與格子上的玩家 / AI 點數標示。
- **牌庫區**：顯示牌背圖片與剩餘牌數。
- **玩家區**：顯示玩家目前手牌，最多 3 張。
- **AI 對手區**：顯示 AI 手牌位置。
- **操作區**：提供抽牌與重新開始按鈕。

### 開發環境

- 作業系統：Windows
- 開發工具：Visual Studio
- 專案類型：C# Windows Forms App
- 啟動入口：`Program.cs` 內 `Application.Run(new Form1());`。

### 執行步驟

1. 以 Visual Studio 開啟本專案的 `.sln` 檔或專案資料夾。

## 程式設計重點

### 1. 牌組與資料結構

程式使用 `Cell` 結構儲存棋盤每格的擁有者與牌值，並以 `List<int>` 管理牌庫、玩家手牌與 AI 手牌；常數包含棋盤大小 9 格、手牌上限 3 張、鬼牌代號 0 等設定，使規則邏輯較易維護。

### 2. 回合與合法行動控制

程式利用 `TurnOwner` 與 `GamePhase` 控制目前是玩家或 AI 回合，以及是否已進入結算狀態；同時透過 `actionTakenThisTurn` 限制每回合只能做一項主要動作，避免連續抽牌或重複操作。

### 3. AI 決策邏輯

AI 會先嘗試完成自己的連線，再考慮是否用鬼牌拆解玩家威脅，接著才是擋線、依位置優先放牌或抽牌，因此不只是單純亂數下法，而是具備基礎策略流程。

### 4. 成線計分

棋盤共有 8 條可判定的線，程式使用 `Lines` 與 `LineNames` 定義橫列、直欄與斜線，再以 `GetValidLines`、`CalculateTotalScore` 與 `BuildScoreReport` 完成結算報告。

### 5. 介面與主題風格

UI 以 Windows Forms 的 `GroupBox`、`TableLayoutPanel`、`PictureBox`、`Panel`、`Label` 與 `FlowLayoutPanel` 組成，並針對玩家、AI、牌庫與棋盤設定不同底色與字體風格，讓畫面更接近卡牌對戰介面。

## 音效設計

專案可加入三段 TTS 語音作為互動回饋：

- `startplay`：新局開始時播放「請抽牌」。
- `winplay`：玩家勝利時播放「恭喜你贏了！」。
- `loseplay`：AI 勝利時播放「電腦獲得勝利，要再挑戰一次嗎？」。  
  這些音效能提升互動感與作品完整度，也符合作業要求中的音效功能。
